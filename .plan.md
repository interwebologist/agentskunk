# DuckDuckGo Search Tool Prompt

You are building a DuckDuckGo search tool for an LLM‑based React agent. The tool must:

- Use the **official DuckDuckGo HTTP API** directly (not a third‑party wrapper).
- Support **web search only**, not news or images.
- Accept a **day filter**
- Default number of results: **5**.
- If a caller requests more than the maximum, respond in JSON explaining that the system can only handle up to `{max}` results and return the default `max` (5) back to the agent.
- For now, ignore advanced operators like `site:`, `intitle:`, or quoted‑phrase handling; just pass the query through as-is.
- assume only basic Latin‑text queries for this LLM agent use case mapped to the duckduckgo http endpoint query
- Return the **raw JSON from DuckDuckGo** as much as possible, since the LLM can interpret it directly.
- Do **not** deduplicate results; return exactly what the API returns.
- On partial‑results cases (fewer than requested), just return the available results; no special error.
- For **network/timeout errors**, enforce a **10‑second timeout** and return a JSON error message saying the search timed out and the agent should “try again later.”
- Do **not** implement logging at this time; only return structured JSON responses and errors to the agent.
- In the README, add **one line** stating that the DuckDuckGo search tool has been added and the agent can interact with it.
- Do **not** change or add content about the `/apply` endpoint in the README; keep that part as it already stands.
- add the tool to the agent.py tool regsistry

Your main responsibilities are:
- Implement the DuckDuckGo tool in `tools/` and wire it into `agent.py` so the React agent can call it.
- Ensure the tool returns JSON‑compatible responses and errors that the agent can understand.
- Enforce the hard limits (default = 5) and explain them clearly in the JSON when exceeded or invalid.
